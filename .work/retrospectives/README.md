# .work/retrospectives/

Phase retrospectives. Written at phase close (by /phase-complete or direction change).

## Naming

`phase-N-retro.md` — e.g. `phase-0-retro.md`.

## Format

Free-form markdown. Recommended sections:
- What we built
- Definition of Done — final status
- Direction changes
- Learning goals — what we learned
- What surprised us
- What we'd do differently
- Adversarial phase review verdict
- Impact on next phase
